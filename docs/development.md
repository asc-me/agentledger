# Development

## Repo layout

```
backend/   FastAPI app; services shared by REST + MCP; provider abstraction; Alembic; tests
web/       Vite + React 19 + TS SPA; Tailwind v4 tokens; TanStack Query; shadcn-style UI
alembic/   (under backend/) migration chain (0001 → head)
docker-compose.yml   postgres(pgvector) + api + web
docs/      documentation (this folder)
```

A deeper map is in [Architecture](ARCHITECTURE.md).

## Backend

```bash
cd backend
uv venv --python 3.12 .venv && source .venv/bin/activate
uv pip install -e ".[dev]"           # add ".[dev,cloud]" for the Anthropic SDK

# zero-infra (SQLite, create_all + seed on boot):
export DATABASE_URL="sqlite:///./dev.db"
# …or Postgres (runs Alembic migrations on boot):
# export DATABASE_URL="postgresql+psycopg://agentledger:agentledger@localhost:5432/agentledger"

uvicorn app.main:app --reload        # http://localhost:8000, OpenAPI at /docs
pytest                               # full backend suite (runs on SQLite, offline)
```

The test suite is fully offline and deterministic (it uses the stub providers and a temp
SQLite DB). Tests are organized by phase in `backend/tests/`.

## Frontend

```bash
cd web
pnpm install
pnpm dev          # http://localhost:5173, proxies /api -> :8000
pnpm test         # Vitest + Testing Library
pnpm typecheck    # tsc -b --noEmit
pnpm build        # tsc + vite build
```

## Migrations (Alembic)

Postgres schema is owned by Alembic and applied on API startup. To change the schema:

```bash
cd backend
# 1. edit the SQLAlchemy models in app/models/__init__.py
# 2. generate a migration against a running Postgres
alembic revision --autogenerate -m "describe the change"
# 3. review the generated file, then apply
alembic upgrade head
```

The runtime applies migrations automatically for Postgres (`app/migrate.py`); SQLite uses
`create_all`, so tests don't need migrations. Custom column types (the pgvector
`EmbeddingType`) may need a hand-check in autogenerated migrations.

## Docker

```bash
docker compose up --build            # full stack
docker compose down                  # stop (add -v to reset the DB volume + reseed)
docker compose logs -f api           # watch API logs (migrations, seed)
```

Resetting the volume (`down -v`) re-runs the full migration chain and reseeds — useful when a
new migration changes the schema.

## Conventions

- **One service layer** — put business logic in `backend/app/services/*`; routers and MCP
  both call it. Don't duplicate logic in a router.
- **Providers** — anything touching an LLM goes through `app/providers` (`Embedder` /
  `ChatModel` / `Extractor`), never a direct SDK call in a route.
- **Frontend data** — go through `lib/api.ts` (authed) or `lib/publicApi.ts` (public), with
  TanStack Query hooks in `lib/queries.ts`. Reuse the design tokens (CSS vars), don't
  hard-code palette values.
